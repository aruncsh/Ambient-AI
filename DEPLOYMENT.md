# 🚀 Deploying Ambient AI to Render

This guide will walk you through deploying the Ambient AI system (Backend + Frontend) to Render using the provided `render.yaml` blueprint.

## 📋 Prerequisites

1.  **Render Account**: Sign up at [render.com](https://render.com).
2.  **MongoDB Atlas**: A free or paid MongoDB database.
    *   Create a cluster at [mongodb.com/atlas](https://www.mongodb.com/atlas).
    *   Get your connection string (e.g., `mongodb+srv://user:pass@cluster.mongodb.net/ambient_ai?retryWrites=true&w=majority`).
3.  **OpenAI API Key**: Required for high-fidelity transcription and SOAP note generation in production.

---

## 🛠️ Step 1: Push to GitHub/GitLab

Render deploys directly from your Git repository. Ensure your code (including `render.yaml`) is pushed to a remote repository.

```bash
git add .
git commit -m "Add Render deployment config"
git push origin main
```

---

## 🏗️ Step 2: Deploy using Blueprint

1.  Log in to the [Render Dashboard](https://dashboard.render.com).
2.  Click **New +** and select **Blueprint**.
3.  Connect your GitHub/GitLab repository.
4.  Render will automatically detect the `render.yaml` file.
5.  You will be prompted to fill in the following **Environment Variables**:
    *   `MONGO_URL`: Your MongoDB Atlas connection string.
    *   `OPENAI_API_KEY`: Your OpenAI API key.
    *   `SECRET_KEY`: (Auto-generated by Render if left blank).
6.  Click **Apply**.

## ⚙️ Configuration Details

### Backend (`ambient-ai-backend`)
- **Type**: Web Service (Docker)
- **Port**: 10000 (Render default)
- **Environment**: Set to use `openai` for Whisper and `llm` for NLP. This is recommended because local processing (Faster-Whisper) requires significant RAM/CPU which is expensive on cloud providers.

### Frontend (`ambient-ai-frontend`)
- **Type**: Static Site
- **Build Command**: `npm install && npm run build`
- **Publish Directory**: `frontend/dist`
- **Auto-linking**: It automatically links to the backend URL via the `VITE_API_URL` variable.

---

## ⚠️ Important Considerations for Production

### 1. File Storage
The current backend stores audio recordings in the `backend/recordings` folder. On Render's Web Service (without a Disk), these files will be **wiped** whenever the service restarts.
- **Solution**: For true production use, you should integrate AWS S3 or Google Cloud Storage for recordings.

### 2. Startup Time
If you are on the **Free Tier**, the backend service will "spin down" after inactivity. The first request after a spin-down may take ~30 seconds to respond.

### 3. WebSocket Support
Render Web Services support WebSockets natively, which is critical for the "Live Stream" feature of Ambient AI.

## 🧪 Post-Deployment Checklist

- [ ] Check the **Events** tab on Render to ensure both services deployed successfully.
- [ ] Open the Frontend URL and try to create a "Critical Emergency" or "Scheduling" encounter.
- [ ] Verify that MongoDB Atlas is receiving data (Collections: `encounters`, `users`, etc.).
- [ ] Test the Microphone button to ensure real-time transcription works via WebSockets.
